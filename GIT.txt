git init -> neues Repository erstellen
git status -> alle neuen oder geänderten Dateien (für einen Commit)

1. working directory erstellen
2. staging area 
git add -> neue commits der staging area hinzufügen + -A = alle
3. repository
git commit -> werden Änderungen eingefügt + -m "Beschreibung"

git log -> gibt alle Änderungen aus

git checkout b275a71cb6f3891c37f801a5e3cd9bb6b5ef6528 -> früheren Zustand ansehen
git checkout master -> zurück zum obersten Commit wechseln


git stash -> merkt Veränderungen, um Branch zu wechseln
git stash apply -> fügt Veränderungen nach Branch-Wechsel ein
